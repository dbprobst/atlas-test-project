Release Notes

Version 1.0

New Features

created these new features for the release.

(12345) New Employee Email Report

We created a new report in the Report tab of the benefits site: Employee Email Report. Now you can easily gather a list of all your employees' email addresses with this report. To enable this report, contact your account manager. Administrators will need help enabling this report. (12345) Enable the New Employee Email Report for an Administrator. Co-source brokers must enable this report before a client can use it. (12345) Enable the New Employee Email Report for an Administrator.

(23456) New Themes for Employee Experience

We added some new color themes for to use on the Employee Experience. Themes come in the following colors:

  • Peppermint - red and green
  • Old Timey - black and sepia
  • Custom - choose your own two colors

Issues

fixed these issues for the release.

  • (34567) Fixed an issue that caused ACA measurements to be incorrect for employees with a last name beginning with the letter L.
  • (45678) Fixed an issue that caused files on shard A to be moved to shared C. Files are now moved to shard B, as expected.

Training

This section contains step-by-step instructions for how to use new or enhanced features.

(12345) Enable the New Employee Email Report

As a employee, you will need to know how to enable the new Employee Email Report for an administrator that wants to use it.

To enable the Employee Email Report:

  1. Click the System tab.
  2. Click Reports in the left menu.
  3. Select the Employee Email Report checkbox.
  4. Click the Save button.
